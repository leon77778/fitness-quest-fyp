# RPGFit

A mobile fitness application that combines role-playing game mechanics with real-world exercise tracking. RPGFit uses AI to generate personalised workout sessions, GPS to track walking activities, and an RPG progression system to keep users motivated.

Built with React Native and Expo, targeting Android.

---

## Features

- **AI-Powered Workouts** — Daily exercise sessions generated by Groq's Llama 3.3 70B model, personalised based on your recent performance history
- **Adaptive Difficulty** — Analyses your last 5 attempts per exercise to automatically adjust targets
- **GPS Walk Tracking** — Real-time walk tracking with AI-generated destination objectives and Google Maps integration
- **XP & Levelling System** — Earn experience points for completing exercises and walks; level up as you progress
- **Weapon Cosmetics** — Unlock 9 tiers of weapon cosmetics from Wooden Stick to Legendary Blade as you level up
- **Streak Tracking** — Tracks current and best consecutive-day streaks to build exercise habits
- **Oracle AI Chatbot** — Context-aware fitness coaching chatbot powered by Groq, personalised to your exercise history
- **Calorie Tracking** — MET-based calorie estimation for both exercises and walks
- **Progress Dashboard** — Calendar view, calorie bar chart, and session history

## Getting Started

### Prerequisites

- Node.js 18+
- npm 9+
- Expo CLI (`npm install -g expo-cli`)
- Android device or emulator

### Installation

1. Clone the repository:
   ```bash
   git clone https://github.com/leon77778/fitness-quest-fyp.git
   cd fitness-quest-fyp
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Set up environment variables. Copy the example config:
   ```bash
   cp src/config.example.js src/config.js
   ```
   Fill in your Supabase URL, Supabase anon key, and Groq API key.

4. Add your Google Maps API key to `app.json`:
   ```json
   "googleMaps": {
     "apiKey": "YOUR_GOOGLE_MAPS_API_KEY"
   }
   ```

5. Start the development server:
   ```bash
   npm start
   ```

6. Scan the QR code with Expo Go on your Android device, or press `a` to open in an emulator.

## Database Schema

Three tables hosted on Supabase PostgreSQL:

- **profiles** — User stats: XP, level, streak, best streak, weight, height, fitness level, equipped weapon
- **sessions** — Individual exercise records: exercise name, type (reps/timer), target, completion status
- **walk_sessions** — Walk records: distance, duration, XP earned, GPS route coordinates, objective

Row-level security ensures users can only access their own data.

## Progression System

| Activity | XP Earned |
|---|---|
| Rep-based exercise | 2 XP per rep completed |
| Timer-based exercise | 1 XP per second |
| Walk (completed) | `floor(distance_metres / 4)` |
| Walk (incomplete) | 0 XP |

**Level formula:** `floor(total_xp / 200) + 1`

**Weapon unlock levels:** Stick (1), Dagger (3), Axe (5), Sword (8), Gold Sword (12), Staff (16), Crystal (21), Trident (27), Legendary (35)

## Key Formulas

**Calorie estimation (MET-based):**
```
Calories = MET × weight_kg × duration_hours
```

Walking MET = 3.5. Exercise MET values vary by exercise type.

**Distance calculation (equirectangular approximation):**
```
x = (lon2 - lon1) × 111320 × cos(lat1)
y = (lat2 - lat1) × 110540
distance = sqrt(x² + y²)
```

Movements below 5 metres are discarded to filter GPS noise.
